### Overall Concept

Task Hub is a canvas-based collaboration tool that reimagines how people work alongside AI. Instead of a chat window bolted onto a task manager, Task Hub treats the canvas itself as the workspace — users create notes and tasks as nodes, draw connections between them, and selectively assign specific nodes to AI agents. The AI's response lands back on the canvas as just another node, keeping human decisions and AI output visible and comparable in the same space.

### Design Goals

The visual design needed to communicate a few things at once:

- **Intentional AI involvement** — the UI should make it easy to see which tasks are "owned" by a user and which have been delegated to AI, so the hand-off feels deliberate rather than automatic.
- **Spatial clarity** — a canvas can get noisy fast. The moodboard and design guidelines focus on a minimal, monochromatic base with a single accent color reserved for AI-touched nodes, so the eye is drawn to what the agent produced.
- **Approachable but technical** — the target user is someone who works with AI tools and wants more control, not a general consumer. The typography and layout lean clean and functional rather than flashy.

### Iterations

The Figma file walks through several layout iterations — from an early dense grid layout that proved too hard to scan, to the current approach of floating cards with explicit connection lines. I tested a few of these locally and the connection-line model won out because it makes the dependency between tasks visible at a glance. The local version I've been testing is what I'm planning to deploy alongside this devlog.
